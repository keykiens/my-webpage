Контрольные вопросы

1. Что такое "асинхронность" в JavaScript, и как она отличается от синхронного выполнения кода?
Асинхронность — это возможность выполнять операции без блокировки основного потока. При синхронном выполнении каждая операция ждёт завершения предыдущей, а при асинхронном — задачи могут выполняться параллельно, и результат обрабатывается позже через колбэки, промисы или async/await.
2. Какой объект в JavaScript используется для работы с асинхронными операциями? Объясните его основные состояния.
Основной объект — Promise. Он может находиться в трёх состояниях:
  • pending — ожидание выполнения;
  • fulfilled — успешно выполнено;
  • rejected — завершено с ошибкой.
3. Чем отличаются async/await от использования метода .then() у промисов? Какие преимущества они дают?
async/await позволяют писать асинхронный код в стиле синхронного, делая его более читаемым. Метод .then() требует вложенных функций и может приводить к «лесенке» кода. Преимущество async/await — простота, наглядность и удобная обработка ошибок через try/catch.
4. Как обрабатываются ошибки в асинхронных функциях с использованием async/await? Приведите пример.
Ошибки обрабатываются через try...catch. Пример: 
async function getData() {
    try {
        const response = await fetch("https://example.com/api");
        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error("Ошибка:", error);
    }
}
5. Что такое "Event Loop" в JavaScript, и как он связан с выполнением асинхронного кода?
Event Loop — это механизм, который управляет выполнением кода, обработкой событий и очередью задач. Он проверяет стек вызовов и очередь задач, выполняя синхронный код сразу, а асинхронные колбэки — когда основной поток освобождается. Благодаря Event Loop JavaScript может быть однопоточным, но при этом эффективно работать с асинхронными операциями.
